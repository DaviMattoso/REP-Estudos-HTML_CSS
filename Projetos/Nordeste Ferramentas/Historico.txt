(Dia 2)

Nota Técnica — Ajustes do Header (Desktop e Mobile)
Objetivo
O objetivo dessa etapa foi construir um header fixo, responsivo e estável, garantindo que:
- o header não cubra o conteúdo (hero)
- não existam espaços em branco gigantes
- o layout funcione corretamente no desktop e no mobile
- o código fique limpo e previsível, sem gambiarras

Problemas encontrados:

1- Header fixo cobrindo o hero
O header usa `position: fixed`, o que remove ele do fluxo normal da página.  
Com isso, o conteúdo (hero) acabava ficando “por baixo” do header.

2- Espaçamentos inconsistentes
Foram identificados vários problemas de espaçamento:

- `padding-top` duplicado no `body`
- `padding` duplicado no `.hero`
- valores muito altos (ex: `250px`) no mobile
- regras se sobrescrevendo dentro do mesmo `@media`

Isso causava comportamentos imprevisíveis:
- em tela cheia ficava errado
- em meia tela ficava certo
- no mobile quebrava tudo

3- Logo quebrando o layout no mobile
O logo estava:
- fora de um agrupamento lógico
- posicionado "acima" do nome da empresa
- deixando o header muito alto no mobile

Resultado: layout “tronxo” e difícil de controlar.

4- Media queries desorganizadas
Havia:
- regras repetidas dentro do mesmo `@media`
- propriedades iguais definidas várias vezes
- dificuldade de entender qual regra realmente estava ativa

Soluções aplicadas:
->Estrutura correta do header
Criei um agrupamento lógico para a marca:
- logo
- nome da empresa
- slogan

O logo passou a ficar "ao lado do nome", tanto no desktop quanto no mobile.  
Isso reduziu a altura do header e estabilizou o layout.

Compensação correta do header fixo
Mantive o header como `position: fixed`, mas corrigi a compensação:
```css
body {
    padding-top: <altura real do header>;
}
